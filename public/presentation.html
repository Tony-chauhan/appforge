Each stage produces a validated intermediate representation (IR), just like a compiler. If a stage fails validation, it retries with error context — not a blind full retry.

Why Multi-Stage?

🎯

Focused Responsibility

Each stage has a single job with a constrained output format. Smaller context = less hallucination.

🔍

Intermediate Validation

We validate between stages. Catch errors early before they propagate downstream.

🔧

Targeted Repair

If Schema Gen fails, we re-run only that stage — not the entire pipeline. Saves cost + time.

📊

Determinism

Structured prompts + JSON mode + temp 0.1 = consistent, predictable outputs across runs.

Cross-Layer Consistency

🔗 API ↔ DB Check

  • Every API endpoint's entity must have a matching DB table
  • If missing → auto-create table with default schema

🔗 UI ↔ API Check

  • Every UI component's dataSource must reference a real API endpoint
  • If missing → flag warning for review

🔗 Auth ↔ All Check

  • Every role used in UI pages and API endpoints must exist in auth config
  • If missing → auto-add role definition

🔗 DB Self-Check

  • Every foreign key must reference an existing table
  • Every column type must be valid PostgreSQL
